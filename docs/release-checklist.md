# 上架检查清单

## 小程序后台

- 注册并认证小程序账号。
- 设置真实 AppID，并确认开发者权限。
- 选择合适服务类目。第一版建议按工具/生活服务类查询工具方向准备材料。
- 填写小程序名称、简介、头像、服务范围。

## 云开发

- 开通云开发环境。
- 在 `miniprogram/utils/config.js` 填写云环境 ID。
- 上传部署 `getWashAdvice` 云函数。
- 上传部署 `searchCity` 云函数。
- 为两个云函数设置环境变量 `QWEATHER_API_KEY`。
- 线上测试云函数返回不含 API key。

## 隐私与权限

- 第一版默认手动城市，不申请定位权限。
- 洗车记录仅本地缓存，不上传身份信息。
- 隐私保护指引中说明：用户主动选择城市、本地缓存洗车日期、天气数据用于生成洗车建议。
- 如果后续加入当前位置，需要申请位置相关接口权限，并在隐私协议中解释用途。

## 天气服务

- 确认和风天气账号、套餐、调用额度。
- 确认数据来源展示符合和风天气要求。
- 生产环境建议使用正式 API host，并配置 `QWEATHER_API_HOST` / `QWEATHER_GEO_HOST` 环境变量。

## 提审前手动测试

- 首次打开：无城市缓存时默认上海，首页可展示。
- 切换城市：选择常用城市后返回首页并刷新。
- 城市搜索：配置云函数后可搜索城市。
- 记录洗车：点击“我今天洗车了”后首页上次洗车变为今天。
- 清空记录：记录页可清空本机记录。
- 天气失败：删除或改错 key 后，首页显示可理解的失败提示。
- 真机预览：检查小屏设备中大字、7 天预报、底部导航没有重叠。

## 发布流程

- 微信开发者工具预览。
- 真机测试。
- 上传代码。
- 在微信公众平台提交审核。
- 审核通过后发布线上版本。
